# 여성기업 필터 설계 (2026-07-20)

## 배경
- 지인 제안: 검색 영역에 "여성기업" 칸이 있으면 좋겠다.
- 데이터 현황(2026-07-20, 공고 1,798건 기준): 기업마당 공식 `target_types` 태그 "여성기업"은 4건뿐.
  제목·요약까지 정밀 키워드(여성기업/여성창업/여성벤처/여성경제인 등)로 확장하면 12건.
- 넓은 매칭("여성" 단순 포함, 29건)은 여성 '고용' 지원(직장문화개선, 채용장려금)과
  단순 언급(농업인 건강검진 등) 노이즈가 절반이라 채택하지 않음. 마이보스 승인됨.

## 결정 사항
1. **UI**: "대상" 칩 줄 맨 앞에 "여성기업" 칩을 항상 고정 표시 (상위 8개 빈도 칩 앞에 prepend).
   토글 동작은 기존 칩과 동일.
2. **매칭**: 여성기업 칩 선택 시 아래 둘 중 하나면 표시.
   - `target_types`에 공식 "여성기업" 태그 포함
   - 제목+요약이 정밀 키워드 정규식 매칭: `여성기업|여성 기업|여성창업|여성 창업|여성기업인|여성벤처|여성경제인|여성CEO|여성대표|우먼|W-벤처` (공백 유연 매칭)
3. **신청 가능 배지**: 공식 태그로 걸린 공고는 기존 규칙 유지.
   키워드로만 걸린 공고는 `eligibility_complete=false` 처리 → "⚪ 원문 확인 필요" 표시.
4. **다른 대상 칩과 조합**: 기존 OR 의미 유지 — 선택된 칩 중 하나라도 해당하면 표시.
   대상 정보가 아예 없는 공고는 (기존 규칙대로) 다른 공식 칩이 선택된 경우에만 "확인 필요"로 노출.
   여성기업 칩만 선택된 경우 키워드/태그 미매칭 공고는 숨김 (노이즈 방지).
5. **수정 범위**: `site/app.js`만 수정. 수집기(collector)·데이터 스키마·GitHub Actions는 변경 없음.
   → 푸시 즉시 GitHub Pages 반영, 새벽 자동 수집과 무관.

## 검증
- 로컬 http 서버 + 브라우저: 여성기업 칩 단독 클릭 시 12건(현재 데이터 기준) 표시 확인.
- 조합 테스트: 여성기업+중소기업 칩, 지역 필터 조합 동작 확인.
- 기존 pytest 47개는 collector 대상이라 영향 없음 (회귀 확인차 1회 실행).
